## Introduction

Parichay Authenticator is a robust authentication framework that offers a variety of multifactor authentication solutions like Token Authentication and Tap Authentication to provide the access to Parichay, Jan Parichay, and others popular services. It also helps in securing the device by providing local device authentication parameters which includes Pattern, Pin, and Fingerprint.

## Functionality

The  application  authenticates  users  via  a  two-step  multi-factor  authentication process where users need to provide a secondary factor of authentication along with the password.

## 1. Tap Authentication:

For  Tap-based  authentication,  users  are  requested  to  tap  a  prompt  sent  by  the authenticator on registered client device. The prompt will ask if the user wants to ' accept ' or ' deny ' the access request for an online application. Following are the benefits of Tap Authentication:

- Easy, quick and convenient real-time authentication solution.
- Authenticates user identity with a single tap.
- Single-tap authentication effectively helps in securing the data in a hassle-free way in addition to meeting the required compliances.
- Access Parichay/JanParichay and integrated services with a single tap.

## 2. Token Authentication:

For  Token-based  authentication,  users  are  requested  to  provide  a  soft  token generated  every  30  seconds  by  the  Authenticator  web  and  installed  application simultaneously. Following are the benefits of Token Authentication.

- This  authentication  factor  adds  a  second  layer  of  security  to  application, network, or service access.
- Provides seamless access to Jan Parichay and integrated services.
- Helps users access the service during the offline mode.

## Multi-factor Authentication

Multifactor  authentication  is  a  security  system  that  utilizes  more  than  one  method  of  user identification for a login or other transaction. This provides an additional layer of security over traditional forms of single sign-on authentication, which require only one type of identification such as a password.

| Multi-factor Authentication Type   | Description                                                                                                                                                                     |
|------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| OTP                                | A one-time dynamic password that is valid only for 10 min                                                                                                                       |
| OTP                                | The web application sends OTP to the registered mobile no./email Id                                                                                                             |
| Tap                                | The web application sends a tap notification to the mobile app with 'Accept' and 'Deny' options. On clicking Accept, the user will login to Parichay/Jan Parichay-Meri Pehchaan |
| Tap                                | The access is denied either on clicking Denied or after 60 seconds whichever is earlier. After denial, the user is redirected to the login page.                                |
| Token                              | Parichay Authenticators app simultaneously generates a token every 30 seconds.                                                                                                  |
| Token                              | Even if the device/mobile has no access to the network, the user can enter this token to log into the Jan Parichay.                                                             |

Note: The Token will start blinking RED in color to aware user about the last 5 seconds of token validity, after that the Token number will be updated.
